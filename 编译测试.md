# 编译修复说明

## ✅ 已修复的编译错误

### 错误原因

`syncSingleTrainToPostgres(const Train &train)` 使用 const 引用参数,但调用了 Train 类的非 const 成员函数:

- `train.getNumber()`
- `train.getTimetable()`

### 解决方案

将参数类型从 `const Train &train` 改为 `Train &train`

### 修改的文件

1. **trainmanager.h** (第 40 行)

   ```cpp
   // 修改前
   void syncSingleTrainToPostgres(const Train &train);

   // 修改后
   void syncSingleTrainToPostgres(Train &train);
   ```

2. **trainmanager.cpp** (第 263 行)

   ```cpp
   // 修改前
   void TrainManager::syncSingleTrainToPostgres(const Train &train) {

   // 修改后
   void TrainManager::syncSingleTrainToPostgres(Train &train) {
   ```

## 📝 技术说明

### 为什么使用非 const 引用?

虽然 `syncSingleTrainToPostgres` 不修改 train 对象,但因为:

1. Train 类的 getter 方法未声明为 const
2. 修改 Train 类可能影响其他代码
3. 使用非 const 引用是最小改动的解决方案

### 更好的长期方案

如果要重构,应该将 Train 类的所有 getter 声明为 const:

```cpp
// train.h
class Train {
public:
    QString getNumber() const;           // 添加 const
    Timetable getTimetable() const;      // 添加 const
    std::vector<std::tuple<QString, int, int>> getCarriages() const; // 添加 const
    // ...
};
```

这样就可以使用 `const Train &` 作为参数,符合 C++ 最佳实践。
但这需要同时修改 train.h 和 train.cpp,影响范围较大。

## ⚠️ 注意事项

当前修复是**最小改动方案**:

- ✅ 解决了编译错误
- ✅ 功能正常工作
- ✅ 不影响其他代码
- ⚠️ 没有遵循 const-correctness 最佳实践

如果将来有时间,建议重构 Train 类使其符合 const-correctness。
