# 测试 Supabase 实时同步功能

## 🧪 测试前准备

### 1. 确保 Supabase 连接正常

检查启动日志是否显示:

```
✅ 成功连接到 Supabase 云数据库!
```

### 2. 准备测试工具

- 选项 A: 打开 Supabase 控制台 (推荐)
  访问: https://supabase.com/dashboard/project/YOUR_PROJECT_ID
  进入 Table Editor -> trains 表

- 选项 B: 启动第二个客户端程序实例

## 📝 测试用例

### 测试 1: 删除列车的实时同步

#### 步骤:

1. 启动程序,进入列车管理页面
2. 记下一个列车的车次号,例如: `G1234`
3. 在程序中删除该列车
4. **立即** (不退出程序) 刷新 Supabase 控制台

#### 预期结果:

- ✅ 程序显示删除成功
- ✅ Qt Creator 输出窗口显示: `[列车管理] ✅ 已从云端删除: G1234`
- ✅ Supabase 控制台中该列车记录消失
- ✅ 整个过程 < 2 秒

#### 失败排查:

如果看到 `[列车管理] 云端删除失败: ...`

- 检查网络连接
- 检查 Supabase 项目是否暂停
- 查看完整错误信息

---

## 📊 性能测试

### 测试同步延迟

使用秒表测量从点击"删除"到 Supabase 控制台更新的时间:

| 操作     | 理想延迟 | 可接受延迟 |
| -------- | -------- | ---------- |
| 删除列车 | < 500ms  | < 2s       |
| 添加列车 | < 800ms  | < 3s       |
| 更新列车 | < 800ms  | < 3s       |

如果延迟超过可接受范围:

- 检查网络质量
- 检查 Supabase 服务器地区 (选择最近的)
- 检查 seat_config JSON 大小 (过大会影响性能)

## 🐛 已知问题 & 解决方案

### 问题 1: "起始站不存在" 警告

**原因**: 车站表中没有该站点记录
**解决**: 先在 StationManager 中添加车站,再添加列车

### 问题 2: 同步失败但不影响使用

**原因**: Supabase 连接超时
**解决**:

- 检查 `railway_debug_pg.ini` 配置
- 验证 Supabase 项目是否激活
- 检查防火墙设置

### 问题 3: 修改车次号后出现重复

**原因**: 旧记录删除失败
**解决**: 重启程序,批量同步会清理
