# Supabase 实时同步修复 - 快速参考

## 🎯 问题总结

删除列车/添加列车等操作后,Supabase 云端数据库更新很慢(只在程序退出时才更新)。

## ✅ 已修复的文件

### 1. `trainmanager.h`

添加了两个新方法:

```cpp
void syncSingleTrainToPostgres(const Train &train);
bool deleteSingleTrainFromPostgres(const QString &trainNumber);
```

### 2. `trainmanager.cpp`

修改了以下方法,实现立即云端同步:

- ✅ `deleteTrain_api()` - 删除列车后立即同步
- ✅ `updateTimetableAndTrainNumberByTrainNumber()` - 添加/更新列车后立即同步
- ✅ `updateSeatTemplateByTrainNumber()` - 更新座位配置后立即同步

新增了两个同步辅助方法的实现(约 140 行代码)。

## 🔧 核心修改原理

### 修改前 (延迟同步)

```cpp
// 删除列车
trains.erase(it);
m_dirty = true;  // 只标记为脏数据
// ❌ 等到程序退出时才调用 saveToPostgres()
```

### 修改后 (立即同步)

```cpp
// 删除列车
trains.erase(it);
m_dirty = true;

// ✅ 立即同步到云端
if (deleteSingleTrainFromPostgres(trainNumber)) {
    m_dirty = false; // 同步成功后清除脏标记
}
```

## 🚀 如何应用到其他 Manager

### 步骤 1: 在 .h 文件中添加方法声明

```cpp
void syncSingleXxxToPostgres(const Xxx &item);
bool deleteSingleXxxFromPostgres(const QString &id);
```

### 步骤 2: 在 .cpp 文件中实现同步方法

参考 `trainmanager.cpp` 中的实现模式:

```cpp
void XxxManager::syncSingleXxxToPostgres(const Xxx &item) {
    if (!railwayPgIsOpen()) {
        qWarning() << "[模块名] 云端连接不可用,跳过同步";
        return;
    }

    QSqlDatabase db = QSqlDatabase::database("railway", false);
    // ... 执行 INSERT ... ON CONFLICT ... DO UPDATE
    // ... 添加日志输出
}
```

### 步骤 3: 在业务方法中调用同步

```cpp
// 添加/更新操作
items.push_back(newItem);
m_dirty = true;
syncSingleXxxToPostgres(newItem);

// 删除操作
items.erase(it);
m_dirty = true;
if (deleteSingleXxxFromPostgres(id)) {
    m_dirty = false;
}
```

## 💡 关键技术点

### 1. UPSERT 语句 (PostgreSQL)

```sql
INSERT INTO trains (...) VALUES (...)
ON CONFLICT (train_number) DO UPDATE SET ...
```

- 如果记录存在则更新,不存在则插入
- 一个语句实现增改两种操作

### 2. 连接健康检查

```cpp
if (!railwayPgIsOpen()) {
    qWarning() << "连接不可用";
    return;
}
```

### 3. 保留兜底机制

即使立即同步失败,程序退出时仍会调用批量 `saveToPostgres()` 作为兜底。

## 📊 性能影响

- **网络请求**: 每次操作增加 1 个 HTTP 请求
- **延迟**: 通常 < 500ms
- **用户体验**: 几乎无感知,但云端实时更新
